# Design System

Data Drift uses a **data-professional-first visual language**, inspired by real-world analytics tools like Excel, Power BI, and CSV workflows, combined with arcade simplicity similar to Chrome Dino.

The goal is not realism — but **instant recognition**.

---

## Design Philosophy

### 1. Familiarity Over Creativity
Every visual element must resemble something a data professional has already seen:

- Excel sheets
- Dashboard charts
- Missing values in tables
- Duplicate rows
- Data type warnings

---

### 2. Arcade Simplicity
Even though the theme is data-driven, visuals must remain:

- Bold
- High contrast
- Minimal detail
- Readable at high speed

---

### 3. Semantic Visual Language
Each visual element represents a real data concept:

| Game Element | Real Data Concept |
|-------------|------------------|
| Player | Data Analyst / Data Engineer Avatar |
| NULL Enemy | Missing values |
| DUP Enemy | Duplicate rows |
| OUTLIER Enemy | Chart anomalies |
| TYPE Enemy | Data type mismatch |

---

## Player Design

### Concept: “Data Node Runner”

Instead of a human character, the player is a **data-processing entity**.

### Visual Traits:
- Glowing circular “data core”
- Minimal geometric body
- Subtle dashboard-like face panel
- Streaming trail effect behind movement

### Animation Style:
- Smooth glide movement
- Soft bounce (arcade feel)
- Directional lean when moving
- Continuous data stream trail

---

## Enemy Design Language

Enemies are designed as **analyst-recognizable UI artifacts**.

---

### NULL (Missing Data)

**Concept:** Empty spreadsheet cell

**Visual Features:**
- Hollow cell box
- Faded interior
- Subtle blinking indicator
- Broken/empty feel

---

### DUP (Duplicate Rows)

**Concept:** Repeated records in a table

**Visual Features:**
- Stacked row layout
- Slight offset duplication
- Highlighted repeated lines
- “copy-paste” illusion

---

### OUTLIER (Anomaly in Data)

**Concept:** Unexpected spike in a chart

**Visual Features:**
- Mini bar chart
- One exaggerated bar spike
- Bright highlight on anomaly
- Slight pulse animation

---

### TYPE ERROR (Schema Mismatch)

**Concept:** Wrong data type in column

**Visual Features:**
- Column-like block
- Warning triangle icon
- Broken or mismatched section styling
- Flickering error indicator

---

## Background Design (Planned)

Future design layer will include:

- Subtle grid (like Excel sheet)
- Faint scrolling data lines
- Dashboard-style overlays
- Minimal neon accents

---

## Effects System

### Movement Effects
- Player trail (data stream effect)
- Enemy pulse animation
- Soft glow around active objects

### Feedback Effects
- Screen shake on collision
- Flash red on error hit
- Combo glow intensification

---

## UI Design

### Score Display
- Minimal top-left HUD
- Shows:
  - Score
  - Combo multiplier

### Style:
- Transparent background
- Monospace or clean sans-serif font
- Subtle glow on high combo

---

## Visual Tone

The game should feel:

- Fast like an arcade runner
- Structured like a data dashboard
- Familiar to analysts
- Minimal like Chrome Dino
- Polished like a modern indie game

---

## Design Goal

To transform abstract data problems into **instantly readable visual gameplay elements**, making data concepts intuitive through interaction rather than explanation.
